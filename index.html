What’s New in this Driver

Important Notes

  • The support for the 12443 port has been withdrawn from HMC 1110 2503060625 onwards. Please use 443 port instead.

  • Support for 7063-CR1 HMCs has been discontinued starting with driver 2504152334.

  • HMC 1110 will not manage Power8 systems. It has been disabled starting with driver 2504252001.

  • It is mandatory to update the eBMC system Firmware to the latest level to get the eBMC system events.

  • Advance automation and monitoring capability (LI 1NZ, 1P1 etc) are not available for P9 systems. This is available for P11 system having latest FSP/eBMC FW. For P10 system, you will need latest FW1060.40 with VET code.

  • Old HMC dashboard is not more supported with this driver level.

How to Determine if a Specific Defect is in this Driver

If you are trying to verify a defect from ClearQuest or RTC, when a fix is submitted for a defect, a comment/message will be added to the defect by the build system displaying the commit message abstract, committer name/email, and the artifacts that include this fix.

This does not however tell you the HMC driver that the defect is included in (because drivers build at a different time). The defect’s fix will be included in the next driver greater than the artifact VRM.

Here is an example below:

Build Message in ClearQuest or Jenkins
Fix fictitious bug in HMC

was recently committed by Doe, John johndoe@in.ibm.com

This commit is available in the following artifacts:
>= pmc-ui-8.1.870.0-1701021545.i386.rpm (1)

Code review and build integration information can be found at
http://powermc-git.aus.stglabs.ibm.com/1
1 pmc-ui-8.1.870.0-1701021545.i386.rpm can be broken down into two parts

Release / Version: 8.1.870.0
Timestamp: 1701021545 (2017 January 2nd 3:45PM)
This fix will now be included in all HMC Drivers for:

  • Release / Version == 8.1.870.0

  • Timestamp >= 1701021545 (greater than or equal to 2017 January 2nd 3:45PM)

Remote Upgrade Using Network Images

  1. If your HMC does not have network connectivity to hmc-driver.rch.tadn.ibm.com, download the appropriate HMC V9 network images and put them on an FTP server. The network images are located in the Austin’s NFS cell under

    x86_64

    HMC-11.1.1110.0-2506141800 x86_64 Network Install Files

    ppc64le

    HMC-11.1.1110.0-2506141800 ppc64le Network Install Files

  2. Verify that the following files have been downloaded: img2a, img3a, base.img, disk1.img, and hmcnetworkfiles.sum

  3. On the HMC, you must first save upgrade data by running the following commands

    To save data to both DVD-RAM and Disk
        mount /media/cdrom
    saveupgdata -r diskdvd (1)
    1 The data will be copied to DVD-Ram for retrieval in case of data loss during the upgrade
    To save to only the disk:
    saveupgdata -r disk
    If your /home directories contain large files such as dumps, pedbg zip files or ISO images, the saveupgdata command will fail if there isn’t enough space to hold them in the upgrade data partition.

    Please cleanup your /home directories and re-issue the command. DO NOT store large data on HMC’s home directories.

  4. Next, run the getupgfiles command to copy the files over to the bootable disk partition on the HMC:

    getupgfiles -r sftp -h <sftp server host> -u <user id> -d <remote directory> (1) (2) (3)
    1 <sftp server host> is the host name or ip address where the network images are copied from step 1 above. Use hmc-driver.rch.tadn.ibm.com if the HMC have network connectivity to NFS.
    2 <user id> and password will be your NFS id and password if you use hmc-driver.rch.tadn.ibm.com
    3 <remote directory> on NFS Network Images

  5. After the files are copied over, run the following command

    chhmc -c altdiskboot -s enable --mode upgrade

    Now the HMC is ready to be rebooted and automatically upgraded with the code copied to the bootable disk partition.

  6. Finally, run the following command to reboot the HMC and start the upgrade

    hmcshutdown -r -t now

Installing HMC on OpenPower Environment

You will be required to select "Install Hardware Management Console" from the petitboot list to start the installation.

Installing using USB Media

  1. Ensure that your USB drive is at least 4GB

  2. Use dd to copy iso to usb. Follow the examples for your OS here:

    Windows

    wiki.archlinux.org/index.php/USB_flash_installation_media#In_Windows

    Mac

    wiki.archlinux.org/index.php/USB_flash_installation_media#In_macOS

  3. Insert USB drive, and power on the machine

Installing via Remote Media from Console Viewer

  1. Login to the BMC web interface (http://<bmc-ip>)

  2. Select Remote Control

  3. Select Console Redirection

  4. Click on Launch Console

  5. In the Java iKVM Viewer, select Virtual Media → Virtual Storage

  6. Under Logical Drive Type select ISO File

  7. Click Open Image and locate the iso on your system

  8. Press Plugin to mount the iso

  9. Power on the machine

    A fast connection between your system and the BMC is recommended. Not having one will extend your installation dramatically.

Installing via Remote Media from BMC web interface

  1. Login to the BMC web interface (http://<bmc-ip>)

  2. Select Virtual Media

  3. Select CD-ROM Image

  4. Fill in the following information

    Share host

    This is the ip of the SMB host, if you are using the hostname, make sure the BMCs DNS is correctly configured.

    Path to image

    This is the SMB path to the system. /<share name>/<rest of path>/<name of iso>.iso

    User (optional)

    This is the username used to login to the smb host

    Password (optional)

    This is the password for the above user

  5. Click Save

  6. Click Mount

  7. Device 1 should now say There is an iso file mounted.

  8. If that does not appear, wait a minute and repeat steps 5 through 7

  9. If you are still having problems, double check the information.

  10. It is common for it to require a few retries for it to work

  11. Power on the machine

    NFS will not work, the BMC does not support SMBv3.

Installing HMC on PowerVM Environment (LPAR)

When installing HMC (ppc64le) on a partition using the HMC’s ISO file, there will be no local graphical console access to the Web UI.

  1. Follow normal procedures for installing via ISO on PowerVM

  2. Once the install completes, the system will boot up to present users with a language selection dialog.

  3. Then a text base license acceptance prompt will appear.

  4. Once the license is accepted you will get a normal login prompt and you can login as hscroot then use the chhmc commands to configure network.

    Make sure the Command Controller is ready to accept commands before issuing command (e.g running lshmc -V until it succeeds).

  5. The following sequence of chhmc commands configure network and opened up ssh as well as remote web ui access on the HMC.

    chhmc -c network -s modify -i eth0 -a <hmc ip address> -nm <hmc network mask> --lparcomm on (1)
chhmc -c network -s modify -h <hmc hostname> -d <hmc domain name> -g <gateway ip> (2)
chhmc -c network -s add -ns <name server> -ds <domain search> (3)
chhmc -c ssh -s enable
chhmc -c ssh.name -s add -a <ip address> (4)
chhmc -c SecureRemoteAccess.name -s add -a <ip address> (4)
hmcshutdown -r -t now (5)
    1 <hmc ip address> is your HMC’s IP adddress
    <hmc network mask> is your HMC’s network mask
    2 <hmc hostname> is your HMC’s hostname
    <hmc domain name> is your HMC’s domain name
    <gateway ip> is the IP Address of your network’s gateway
    3 <name server> is your network’s name server address
    <domain search> is the domains you would like the HMC to search on
    4 To allow all ip addresses access, use: -a 0.0.0.0 -nm 0
    5 A reboot needs to be performed

  6. Installation and network configuration is complete.

Installing KVM on Red Hat Enterprise Linux 7.3 (ppc64le)

RHEL 7.3 on ppc64le architecture does not provide the needed rpm to use KVM. Instead, you will need to obtain the necessary rpm files from Red Hat Virtualization Host (RHVH) 4.1. This will allow vHMC on Power to be used with KVM on RHEL.

  • Make sure you have the following packaged installed via yum:

 libvirt-daemon libvirt-daemon-config-network libvirt-daemon-config-nwfilter
 libvirt-daemon-driver-lxc libvirt-daemon-driver-qemu
 libvirt-daemon-driver-nwfilter libvirt-daemon-driver-interface
 libvirt-daemon-driver-secret libvirt-daemon-driver-storage
 libvirt-daemon-driver-network libvirt-daemon-driver-nodedev
 augeaslibibverbs-devel librdmacm-utils librdmacm ipxe-roms-qemu.noarch
 libfdt pulseaudio-libs libusbx-devel libusb gperftools-libs SLOF
 seavgabios-bin libcacard usbredir libvirt

  • Obtain the following rpm files from ftp3.linux.ibm.com/files/redhat/beta_cds/RHEV-4.1-RC-MAR-28-2017/Management-Agent-Power-7/ppc64le/os/Packages/

  qemu-img-rhev
  qemu-kvm-common-rhev
  qemu-kvm-rhev
  qemu-kvm-tools-rhev
The link above may change. At this time RHVH 4.1 has only gone release candidate. It may go to more release candidate iterations before GA.

  • Install these rpm files using rpm or yum install.

  • If you run virt-manager as root, edit the file /etc/libvirt/qemu.conf and uncomment the user and group values to root.

     #user = root  -> user = root
     #group = "root" - > group = "root"

  • Restart libvirtd service

     systemctl restart libvirtd.service

HMC Development / Debugging

Unlocking HMC will create the user hscpe. You can log into an unrestricted shell on the HMC as hscpe with password of abc1234 .

This will allow HMC development to log on to your HMC and debug.

Unlocking HMC with CD

  1. Download the ISO image Open Cert

  2. Create a CD with the ISO image

  3. On HMC GUI Navigation Area
    Updates → Update HMC → Install Corrective Service
    task on the HMC to install the code on this CD.

  4. Reboot the system.

  5. When the system is rebooted, press Ctrl-Alt-F1 to access the console.

  6. The console will have a login prompt.

  7. To access the HMC GUI, press Ctrl-Alt-F2.

Unlocking HMC with Corrective Service

  1. On HMC GUI Navigation Area
    HMC Management → Console Management → Update Hardware Management Console

  2. Select to download the corrective service file from remote server

  3. Enter Remote Server name: hmc-driver.rch.tadn.ibm.com

  4. Enter your NFS User ID

  5. Enter your NFS Password

  6. Enter the Remote directory: /dev/OpenCert/

  7. In the list of corrective service packages shown, select HMC-11-1-OpenCert.iso

  8. Enter Next, then Finish.

A reboot is not required

Unlocking HMC with CLI

  1. Login via SSH as hscroot user

  2. Perform the following command

    updhmc -t nfs -l /HMCReleases -h hmc-driver.rch.tadn.ibm.com -f /OpenCert/HMC-11-1-OpenCert.iso

Enabling HMC Features for a Future Release

This should only be used by test and development.

DO NOT enable this in a production environment (or on a customer’s HMC)

  1. Unlock HMC with Open Cert

  2. Login via SSH as hscpe user

  3. Perform the following command

    sudo sh -c 'echo 0.0.0.0 /etc/hmc-release.d/override' (1)
    1 Where 0.0.0.0 is the version of HMC you would like to enable features for

  4. Reboot the HMC

    hmcshutdown -r -t now
The HMC version will consist of 4 segments

:: Version.Release.Maintenance.Fix ::: Version: 8, 9, or 10
Release = 1 (may differ, ask Future HMC Release Lead)
Maintenance = HMC Internal Release (IE 870, 910, 920…​)
    xor Service Pack (IE 871, 912, 921…​)
Fix = 0 - None, 1 - MPTF, 2 - FixPack, …​

Examples:

:: 8.1.871.0 - HMC 870 Service Pack 1 :: 8.1.871.1 - HMC 870 Service Pack 1 - First Monthly PTF :: 9.1.910.0 - HMC 910 GM (Base) :: 9.1.920.1 - HMC 920 Mandatory PTF

Returning HMC to Original Feature Function

  1. Login via SSH as hscpe user

  2. Perform the following command

    sudo rm /etc/hmc-release.d/override

  3. Reboot the HMC

    hmcshutdown -r -t now

Configuration Help for HMC

Configuring RAID 1 on the HMC (x86_64)

  1. Ensure both hardfiles are properly installed in the HMC appliance. The hardfiles must be of the same type (both SATA or both SAS).

  2. Reboot or Power On the HMC Appliance at the console.

  3. At some point in the boot, you will be prompted to press F1 to go to setup

  4. Select System Settings

  5. Select LSI MegaRAID <…​> Configuration Utility

  6. Select Virtual Drive Management

  7. Select Create Configuration

  8. Scroll down and select Select RAID Level and choose RAID1

  9. Scroll down and select Select Drive

  10. Hit the space bar against each drive’s checkbox to get them selected

  11. Select Apply Changes

  12. Select OK

  13. Select Save Configuration

  14. Hit the space bar against the Confirm checkbox and select OK

  15. You may ignore the following message

    Ignore Message
      Additional Virtual Drives cannot be created due to insufficient capacity or absence of configurable Physical Driver.

  16. Hit the Escape Button until you exit the Setup Utility

  17. Press Y to save and exit the Setup Utility

  18. On reboot, you should notice on the console the following message

    Validation Message
    Virtual drive(s) found on the host adapter

    which validates your Virtual drive was properly configured.

Configuring RAID 1 on the HMC (ppc64le - Stratton Boxes)

  1. Download MVCLI CLI from SuperMicro ftp.supermicro.com/driver/P8DTU/Onboard%20Marvell%209230/Tools/mvcli

  2. Copy mvcli to Stratton when running in petitboot through USB or Network.

  3. Execute the following commands:

    modprobe sg (1)
chmod +x mvcli (2)
./mvcli info -o vd (3)
./mvcli info -o hba (4)
./mvcli create -o vd -r1 -n "HMC Disk" -d 0,1 (5)
./mvcli info -o vd (6)
    1 Load sg module in petitboot
    2 Make mvcli binary executable
    3 Command to display Virtual Disks
    4 Command to retrieve Disk Adapter IDs
    5 Command to create Virtual Disk:
    create -o vd → Virtual Disk
    -r1 → Raid 1 Configuration
    -n "HMC Disk" → Disk Name
    -d 0,1 → Disk adapter Ids
    6 Command to display Virtual Disk and verify that Disks are in Raid 1

Configuring RAID 1 on the Power 9 HMC (ppc64le - Mowgli Boxes)

  1. Boot the HMC to petitboot and select the option to exit to petitboot shell.

  2. Execute the following commands:

    /bin/arcconf create 1 logicaldrive name "HMC Disk" max 1 0,0 0,1 (1)
/bin/arcconf getconfig 1 ld (2)
/bin/arcconf setcontrollerparam 1 spareactivationmode 1 (3)
    1 Create the RAID1 logical drive
    2 Display the logical drive to confirm that it was created
    3 Enable the automatic rebuild of the logical drive
